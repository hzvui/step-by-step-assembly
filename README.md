Данный набор компонентов (`AssemblyGuide`, `InfoWindow`, `TabletDisplay`) реализует **интерактивную систему пошаговой сборки модели** с поддержкой двух режимов установки деталей: **ручного** (перемещение объектом в VR или на ПК) и **автоматизированного с помощью крана**. Система предназначена для обучающих приложений, тренажёров и симуляторов, где требуется наглядно демонстрировать процесс сборки с контролем правильности выполнения каждого шага.

## Основные возможности

### 1. Гибкое определение шагов сборки
- Каждый шаг (`AssemblyStep`) настраивается в редакторе и содержит:
  - Текстовое описание действия.
  - Целевой объект (`ObjectMeshStatic`), участвующий в шаге.
  - Начальную и целевую позиции (мировые координаты).
  - Допуски по положению (`PositionTolerance`) и углу поворота (`RotationTolerance`) для контроля точности установки.
  - Флаг `isKran` – выбор способа установки: ручная или с помощью крана.
  - Для крана: ссылки на кнопки подъёма/опускания и порог срабатывания (`ThresholdZ`).
  - Путь к изображению (`texturePath`) – иллюстрация шага (опционально).

### 2. Визуальные подсказки
- При активации шага создаётся полупрозрачный **«призрачный» объект (ghost)** красного цвета, показывающий идеальное конечное положение детали.
- В ручном режиме при поднесении детали к цели цвет призрака меняется (настраиваемая индикация), сигнализируя о готовности к фиксации.
- Для крана объект плавно опускается при нажатии на соответствующие кнопки; по достижении целевой высоты он автоматически фиксируется.

### 3. Два режима установки
- **Ручной режим** (`isKran = false`):  
  Деталь перемещается с помощью системы манипуляции (`DrugObject` и `VRTransformMovableObject`). Когда деталь оказывается в пределах допусков от призрака, она может быть зафиксирована (через вызов `CompleteCurrentStep`). При фиксации деталь «примагничивается» к целевой позиции, её физика отключается, и происходит переход к следующему шагу.

- **Крановый режим** (`isKran = true`):  
  Деталь управляется вертикальным перемещением с помощью двух физических кнопок (в VR) или клавиш (на ПК). При опускании до целевой высоты (с учётом возможного смещения всей конструкции через компонент `Test_key_j_and_k`) деталь автоматически фиксируется в правильном положении и повороте, и шаг завершается.

### 4. Интеграция с интерфейсом
- **InfoWindow** – создаёт окно с инструкцией на основном экране (GUI), отображая текст текущего шага.
- **TabletDisplay** – выводит ту же инструкцию на планшет (`ObjectGui`), что удобно в VR.
- Если задан `texturePath`, изображение шага загружается и отображается поверх интерфейса (например, схема или фотография).

### 5. Лёгкая расширяемость
- Массив шагов (`Steps`) можно свободно редактировать в инспекторе: добавлять новые шаги, удалять существующие, менять порядок.
- Все параметры шагов доступны для тонкой настройки прямо в редакторе, без необходимости писать код.
- Благодаря модульной архитектуре компоненты могут быть легко переиспользованы в разных сценах и проектах.

### 6. Поддержка VR и ПК
- Система одинаково хорошо работает как в виртуальной реальности (с физическими кнопками для крана и захватом объектов контроллерами), так и на обычном ПК (с клавиатурой и мышью).

## Назначение
Данный инструмент идеально подходит для создания:
- Обучающих симуляторов сборки узлов и механизмов.
- Интерактивных инструкций по эксплуатации оборудования.
- Тренажёров для отработки навыков монтажа с контролем правильности.
- Демонстрационных сцен, где требуется поэтапно показать процесс сборки.

Компоненты спроектированы так, чтобы минимизировать необходимость программирования и позволить дизайнерам и методистам самостоятельно создавать сложные сценарии сборки прямо в редакторе UNIGINE.
